# VPIndex

Плагин для Minecraft Paper 1.21+, который отслеживает активность игроков и вычисляет **индекс активности** — комплексную метрику, отражающую реальную вовлечённость и активность игрока.

## Формула

```
Индекс = Очки / (Наигранное время − AFK время)
```

За каждое действие игрока (слом блока, убийство моба, крафт и т.д.) начисляются очки. Система демпфирования снижает очки при повторении одних и тех же действий, чтобы боты и фармеры не накручивали статистику.

## Зависимости

| Зависимость | Тип | Описание |
|---|---|---|
| PlaceholderAPI | softdepend | Плейсхолдеры |

## Команды

| Команда | Описание |
|---|---|
| `/index` | Справка |
| `/index reload` | Перезагрузка конфигов |
| `/index player <ник> [дни] [смещение]` | Статистика игрока |
| `/index top <index/time/points> <best/worst> <кол-во>` | Топ игроков |
| `/index average <index/points/time> [дни]` | Среднее значение |
| `/index sort <best/worst> <кол-во> <placeholder>` | GUI-меню с топом |
| `/index addpoint <ник/#all/#online> <кол-во>` | Добавить очки |
| `/index delete <ник/#all/#online> [дни]` | Удалить статистику |
| `/index debug <ник/#all>` | Отладка начисления очков |
| `/index hologram create/remove/kill/restore <имя>` | Управление голограммами |

Все подкоманды требуют право `vpindex.admin`.

## PlaceholderAPI

| Плейсхолдер | Описание |
|---|---|
| `%activity_index%` | Индекс активности |
| `%activity_time%` | Наигранное время |
| `%activity_afktime%` | Время AFK |
| `%activity_activetime%` | Активное время |
| `%activity_points%` | Очки |
| `%activity_afk%` | Статус AFK |

Плейсхолдеры поддерживают фильтрацию по периоду: `%activity_time_30%` - за 30 дней, `%activity_time_30_7%` - за 30 дней, начиная с 7 дней назад.

## Возможности

- **25 типов событий** - блоки, мобы, крафт, чат, движения и многое другое
- **Демпфирование** - очки снижаются при повторении однотипных действий
- **AFK-система** - автоматическое определение бездействия
- **Голограммы** - таблицы лидеров через TextDisplay
- **GUI-меню** - список игроков с головами
- **Whitelist** - отслеживание только игроков из вайтлиста
- **SQLite** - данные хранятся локально в файле `database.db`
- **Логирование** - все действия администратора записываются в файлы
